# Strategy Framework

本系統把先前所有策略與 finance skills 的原則整合成七層決策框架。

## 1. 產業催化

- 來源：新聞、輿情、政策、交易所公告。
- 目的：先找有利基與催化事件的產業，不從全市場盲選。
- 風控：證據數不足或來源品質弱時降低信心。

## 2. 財務品質

- 營收年增、營業利益率、自由現金流率、負債權益比。
- 對應 finance skills：不只看題材，也看現金流、毛利/營利能力、槓桿與財務韌性。

## 3. 估值與風險

- 本益比過高扣分。
- 風險收益比低於 2:1 時降評，低於 1:1 時限制為候選觀察。
- 對應 finance skills：用 valuation 與 downside risk 當作交易前置條件。

## 4. 日線蠟燭圖

- 來自《日本蠟燭圖技術新解》：錘子線、流星線、十字星、吞噬、刺穿、烏雲蓋頂、啟明星、黃昏星、窗口、迭創新高/低、差異指數、三線反向突破。
- 原則：位置比型態重要，型態必須搭配支撐、壓力、趨勢與超買超賣。

## 5. 1H 市場結構

- 來自小紅書影片策略。
- HH/HL：只優先找多方機會。
- LH/LL：只優先找空方或減碼機會。
- 中性：不主動追價。

## 6. 5M 進場觸發

- 掃下方流動性後收回：多方候選。
- 掃上方流動性後收回：空方/減碼候選。
- IFVG：等待回測缺口不破或受阻後再進場。

## 7. 交易計畫

- 進場：收盤確認或 5M IFVG 回測確認。
- 停損：放在掃流動性高低點外，或多/空型態失效點。
- 出場：前一交易日高低點、前高/前低、三線反向突破反轉、十字星/紡錘線動能衰竭。
- 部位：盤中未確認僅試單，收盤確認才加碼。

## 8. 模型訓練資料護欄

- 機器學習分數只能作為排序輔助，不可單獨產生買進建議。
- 訓練前先清洗資料，排除黑天鵝、普漲行情、漲跌停、暫停交易、重大事件與明顯資料錯誤。
- 使用時間序列切分，不使用隨機切分。
- 不在同一段最終測試資料上反覆調參。
- 若模型只在普漲行情有效，降低模型權重。
- 詳細規則見 `docs/MODEL_TRAINING_DATA_GUARDRAILS.md` 與 `stock_signal_system/training_data.py`。
