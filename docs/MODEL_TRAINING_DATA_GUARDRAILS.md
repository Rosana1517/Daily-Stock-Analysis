# 模型訓練資料護欄

這份規則納入使用者提供圖片中的核心觀點：模型好壞不只取決於怎麼訓練，更取決於訓練資料是否洗乾淨。黑天鵝事件與普漲行情若大量混入訓練集，會讓模型在訓練時看起來很漂亮，但在一般市場環境驗證時失真，形成過擬合。

## 核心原則

1. 先清洗資料，再訓練模型。
2. 黑天鵝與普漲行情不當作一般可重複訊號。
3. 驗證集與測試集必須用時間序列切分，不使用隨機切分。
4. 特徵只能使用當時已知資料，避免偷看未來。
5. 模型輸出只能作為候選排序與風險提示，不單獨產生買進建議。

## 必須排除或降權的資料

- 指數單日大漲或大跌超過門檻，例如加權指數單日絕對報酬超過 7%。
- 橫截面普漲行情，例如 80% 以上股票上漲且中位數漲幅超過 3%。
- 黑天鵝事件前後窗口，例如事件前 1 天到事件後 3 天。
- 漲跌停、暫停交易、重大處置、資料錯誤、異常成交量。
- 政策、戰爭、金融危機、疫情、重大天災等不可重複事件。

## 在本專案中的落點

- `stock_signal_system/training_data.py`
  - `detect_dirty_training_dates`: 偵測黑天鵝與普漲日期。
  - `purge_date_window`: 擴大排除事件前後窗口。
  - `filter_training_samples`: 移除髒樣本與帶有排除標籤的樣本。
- `tests/test_training_data.py`
  - 鎖定普漲、黑天鵝、事件窗口與標籤排除規則。

## 未來接入模型訓練時的順序

```text
原始日線/1H/5M 資料
→ 檢查缺值、重複、異常 OHLCV
→ 偵測黑天鵝與普漲日期
→ 排除事件窗口與特殊標籤樣本
→ 產生特徵與標籤
→ 依時間序列切分 train / validation / test
→ 只在 train 上調參
→ 在 validation 選模型
→ 最後只用 test 做一次最終驗證
→ 與規則式策略共同投票，不單獨決策
```

## 防過擬合檢查

- 訓練績效遠高於驗證績效時，視為過擬合。
- 只在普漲行情有效、盤整或回檔失效時，降低模型權重。
- 特徵數量過多但樣本數不足時，降低模型複雜度。
- 每次改策略後保留獨立測試期，不用同一段資料反覆調參。
- 報告中若模型訊號來自低品質樣本，必須標示「模型信心不足」。

## 對每日選股報告的影響

目前系統仍以規則式策略為主。若未來加入機器學習模型，模型分數只能作為以下用途：

- 候選股排序加權。
- 風險旗標。
- 題材與技術面共振確認。
- 排除過度依賴普漲行情的假訊號。

不允許模型因為訓練集表現漂亮，就直接覆蓋產業、基本面、技術面與風險收益比規則。
